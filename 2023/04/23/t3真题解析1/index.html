铁三线下赛真题解析b站视频

题型

  • 个人赛：逆向及pwn
  • 企业赛：渗透以及安全加固
  • 数据赛：流量分析，取证

个人赛

讲的东西挺入门的，略过

数据赛

题目环境

  • Pcap包：或会伴随日志文件给出
  • Shell接口：需要从服务器中挖掘信息
  • Pcap包和Shell接口二合一

攻击者的操作：

  1. 通过端口攻入server
  2. 内网扫描
  3. 设置代理，进行内网攻击

任务

在内网中存在端口镜像机制（将交换机所有端口的流量镜像到一个端口），我们需要通过审查其中的流量对黑客的攻击进行溯源，从而答题。

工具

  • Pcap包：Wireshark
  • 日志：Notepad++、ApacheLogsViewer(Windows)；Vim(linux)

Wireshark的使用

界面介绍就不多看了，重点在于捕获和过滤器。

捕获

在捕获设置处可以设置，捕获到多少分组/内存/时间就会保存捕获的信息到文件。

捕获的接口也可以设置。

过滤器

背景为红色表示命令错误、黄色表示可能出错、绿色表示正确。

常用命令
  • 过滤IP：ip.addr==……或ip.addr eq …… （addr/src/dst）
  • 过滤协议：http tcp ftp …
  • 过滤http请求方法：http.request.method==POST/GET
  • 过滤URL中的关键字：http.request.uri contains ……
  • 过滤http请求状态码：http.response.code==404(…)
  • 连接符：&& || and or
Trick
  • 目录扫描：大量404请求
  • sql注入：大量select … from 关键字请求
  • 端口扫描：连续一个ip的多端口请求或多个端口的几个相同端口请求
从pcap包中提取文件

  1. 文件 --> 导出对象 --> http

  2. 定位分组位置 --> 右键 --> 追踪流 --> TCP流 --> 选择原始数据 --> save as

日志分析

Vim

  • 统计文件中字符串出现的次数 ：:%s/字符串//ng
  • 统计m行到n行字符串出现的次数：:m,ns/字符串//gn

在apachelog中找到访问次数最多的10个IP：

awd'{print $1}' apache_log |sort |uniq -c\sort -nr\head -n 10

常见关键字：Login admin manage edit upload eval assert POST tunnel select alert 404

DHCP（动态主机设置协议）（过滤器中是bootp）

  1. Discover：client请求发现server
  2. Offer：server分配ip和网关之类的信息
  3. Request：client回应server
  4. ACK：server确认信息

TCP（文件传输协议）

用于在网络上进行文件传输，使用C/S架构，属于应用层。FTP一般运行在20和21两个端口，20用于传输数据流，21用于传输控制流。
